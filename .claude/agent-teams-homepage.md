# 에이전트 팀 (Agent Teams)

여러 Claude Code 인스턴스를 조율해 병렬 작업을 수행하는 실험적 기능.

## 왜 필요한가?

복잡한 작업은 한 명의 에이전트만으로 처리하기 어렵다. 에이전트 팀은 다음 방식으로 이 문제를 해결한다.

- 한 세션이 **리더**가 되어 작업을 조율
- 여러 **팀원**이 각자 독립된 컨텍스트에서 병렬 작업 수행
- 팀원끼리 **직접 메시지**를 주고받으며 협업 가능

> 요구사항: Claude Code v2.1.32 이상

---

## 핵심 장점

### 병렬 탐색
여러 관점에서 동시에 문제를 탐색할 수 있다.

### 역할 분리
UX, 아키텍처, 테스트, 보안 등 역할별로 분업할 수 있다.

### 직접 협업
Subagents와 달리 팀원끼리 직접 통신할 수 있다.

### 복잡한 작업에 적합
연구, 검토, 경쟁 가설 디버깅, 모듈별 개발에 유리하다.

---

## 언제 쓰면 좋은가?

- 연구 및 리뷰를 여러 관점에서 동시에 진행할 때
- 새 기능이나 모듈을 역할별로 나눠 개발할 때
- 버그 원인을 여러 가설로 병렬 조사할 때
- 프론트엔드 / 백엔드 / 테스트를 교차 조율해야 할 때

## 이런 경우엔 비추천

- 순차적으로 끝나는 단순 작업
- 여러 팀원이 같은 파일을 동시에 수정해야 하는 작업
- 의존성이 지나치게 많은 작업
- 토큰 비용이 중요한 일상 작업

---

## Subagents와 차이

| 항목 | Subagents | Agent Teams |
|---|---|---|
| 통신 | 메인 에이전트에게만 보고 | 팀원끼리 직접 메시지 가능 |
| 조율 | 메인 에이전트 중심 | 공유 작업 목록 기반 |
| 비용 | 낮음 | 높음 |
| 적합한 일 | 결과만 필요한 집중 작업 | 협업과 토론이 필요한 복잡한 작업 |

**기준:** 서로 통신이 필요하면 Agent Teams, 아니면 Subagents.

## 시작 방법

### 1) 기능 활성화

`settings.json`

```json
{
  "env": {
    "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
  }
}
```

또는 환경 변수:

```bash
export CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1
```

### 2) 팀 생성

Claude에게 자연어로 팀 구성을 설명하면 된다.

```text
CLI 도구 설계 탐색을 위한 에이전트 팀을 만들어줘:
- UX 담당 팀원 1명
- 기술 아키텍처 담당 팀원 1명
- 악마의 변호인 역할 팀원 1명

각자 독립적으로 문제를 탐색하고 서로 발견을 공유해.
```

### 3) 팀 제어

- `Shift+Down`: 팀원 순환
- `Enter`: 팀원 세션 보기
- `Escape`: 현재 턴 중단
- `Ctrl+T`: 작업 목록 보기

## 대표 사용 예시

### 병렬 코드 리뷰
- 보안 검토자
- 성능 검토자
- 테스트 커버리지 검토자

### 경쟁 가설 디버깅
- 서로 다른 가설을 각 팀원이 병렬 검증
- 반박과 재검토를 통해 더 빨리 수렴

### 모듈별 병렬 개발
- 인증
- DB 레이어
- API
- 테스트

각 팀원이 파일 소유권을 나눠 작업하면 충돌을 줄일 수 있다.

---

## 운영 팁

### 충분한 컨텍스트 제공
팀원은 리더의 대화 기록을 상속하지 않으므로 생성 프롬프트에 필요한 배경을 직접 넣어야 한다.

### 팀 크기 조절
보통 3~5명부터 시작하는 편이 효율적이다.

### 작업 단위 쪼개기
너무 작아도 비효율이고, 너무 크면 낭비가 커진다. 함수, 테스트 파일, 리뷰처럼 경계가 명확한 단위가 적당하다.

### 파일 충돌 피하기
같은 파일을 여러 팀원이 편집하지 않도록 분리한다.

### 리더가 기다리기
리더가 먼저 구현에 들어가지 않고 팀원 결과를 충분히 수집해야 한다.

---

## 제한 사항

- 세션 재개 불가
- 느린 종료 가능
- 세션당 한 팀만 관리 가능
- 중첩 팀 불가
- 분할 창 모드는 일부 터미널에서 미지원

---

## 빠른 요약

- Agent Teams는 **병렬 협업용**
- **복잡한 작업**에서 효과적
- 팀원끼리 **직접 통신 가능**
- 단순 작업엔 **Subagents가 더 적합**
- 시작은 **3~5명 팀 + 명확한 역할 분리**가 무난

---

더 자세한 운영 방식, 훅, 아키텍처, 문제 해결 내용은 상세 가이드를 참고하면 된다.
